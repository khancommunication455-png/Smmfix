# SMM Elite - Production-Ready Social Media Marketing Panel\n\n## Overview\n\nSMM Elite is a complete SaaS platform for managing social media services. It handles user authentication, orders, payments, service integration, and comprehensive admin management.\n\n## Features\n\n- **User Management**: Registration with referral system, profile management, account status control\n- **Service System**: Multiple SMM providers, automatic service sync, category management\n- **Orders**: Create, track, and manage service orders with real-time status updates\n- **Payments**: Stripe & PayPal integration, manual deposit approval, referral commissions\n- **Analytics**: Revenue tracking, order analytics, user statistics\n- **Support**: Ticket system with admin replies and resolution tracking\n- **Admin Panel**: Complete management dashboard with provider sync, user management, logs\n- **API Integration**: Automatic provider API integration with error handling & retry logic\n- **Queues**: Background job processing for orders and notifications\n- **Logging**: Comprehensive activity, payment, and provider API logging\n\n## Tech Stack\n\n- **Framework**: Laravel 10+\n- **Database**: MySQL 8.0+\n- **Queue**: Redis or Database\n- **Frontend**: Blade templates + Tailwind CSS + Alpine.js\n- **Payment**: Stripe SDK + PayPal SDK\n- **Hosting**: Railway (recommended) or any Laravel-compatible host\n\n## Prerequisites\n\n- PHP 8.2+\n- Composer\n- Node.js 18+\n- MySQL 8.0+ or PostgreSQL\n- Redis (optional, for queues)\n\n## Installation\n\n### 1. Clone Repository\n\n```bash\ngit clone <repository-url>\ncd final_panel\n```\n\n### 2. Install Dependencies\n\n```bash\n# PHP dependencies\ncomposer install\n\n# Node.js dependencies\nnpm install\n```\n\n### 3. Environment Setup\n\n```bash\n# Copy environment file\ncp .env.example .env\n\n# Generate application key\nphp artisan key:generate\n```\n\n### 4. Configure .env\n\nEdit `.env` with your configuration:\n\n```env\nAPP_NAME=\"SMM Elite\"\nAPP_ENV=production\nAPP_DEBUG=false\nAPP_URL=https://yourdomain.com\n\n# Database\nDB_CONNECTION=mysql\nDB_HOST=127.0.0.1\nDB_PORT=3306\nDB_DATABASE=smmelite\nDB_USERNAME=root\nDB_PASSWORD=your_password\n\n# Payment Gateways\nSTRIPE_PUBLIC_KEY=pk_live_...\nSTRIPE_SECRET_KEY=sk_live_...\nSTRIPE_WEBHOOK_SECRET=whsec_...\n\nPAYPAL_MODE=live\nPAYPAL_CLIENT_ID=...\nPAYPAL_SECRET=...\n\n# Email\nMAIL_MAILER=smtp\nMAIL_HOST=smtp-relay.brevo.com\nMAIL_PORT=587\nMAIL_USERNAME=...\nMAIL_PASSWORD=...\nMAIL_FROM_ADDRESS=noreply@smmelite.com\n```\n\n### 5. Database Setup\n\n```bash\n# Run migrations\nphp artisan migrate\n\n# Seed initial data (optional)\nphp artisan db:seed\n```\n\n### 6. Build Assets\n\n```bash\n# Development\nnpm run dev\n\n# Production\nnpm run build\n```\n\n### 7. Start Queue Worker (if not using sync driver)\n\n```bash\nphp artisan queue:work\n```\n\n### 8. Start Development Server\n\n```bash\nphp artisan serve\n```\n\nAccess at: `http://localhost:8000`\n\n## Railway Deployment\n\n### Prerequisites\n\n- GitHub account with repository\n- Railway account (https://railway.app)\n\n### Steps\n\n1. **Connect Repository**\n   - Go to Railway dashboard\n   - Click \"New Project\" → \"Deploy from GitHub\"\n   - Select your repository\n\n2. **Add Services**\n   - Add MySQL service\n   - Add Redis service (for queues)\n\n3. **Configure Environment**\n   - In Railway, add variables from Production `.env`\n   - Set `APP_ENV=production`\n   - Set `APP_DEBUG=false`\n   - Configure database credentials from MySQL service\n   - Configure Redis from Redis service\n\n4. **Deploy**\n   - Railway automatically detects `Procfile`\n   - Deploys web dyno, worker, and scheduler\n\n5. **Run Migrations**\n   - After first deploy, run in Railway terminal:\n\n```bash\nphp artisan migrate --force\nphp artisan db:seed --force\n```\n\n### Railway Procfile Format\n\nThe included `Procfile` runs:\n- **web**: Apache server serving the application\n- **worker**: Queue worker for background jobs\n- **scheduler**: Cron scheduler for scheduled tasks\n\n## Payment Gateway Setup\n\n### Stripe\n\n1. Get keys from https://dashboard.stripe.com/apikeys\n2. Set `STRIPE_PUBLIC_KEY` and `STRIPE_SECRET_KEY` in `.env`\n3. Set webhook endpoint in Stripe dashboard:\n   - Endpoint: `https://yourdomain.com/api/webhooks/stripe`\n   - Select events: `charge.completed`, `charge.failed`, `charge.disputed`\n4. Copy webhook secret to `STRIPE_WEBHOOK_SECRET`\n\n### PayPal\n\n1. Get credentials from https://developer.paypal.com/dashboard\n2. Set `PAYPAL_CLIENT_ID` and `PAYPAL_SECRET` in `.env`\n3. Set `PAYPAL_MODE=live` for production\n4. Configure webhook in PayPal dashboard:\n   - Endpoint: `https://yourdomain.com/api/webhooks/paypal`\n\n## Admin Setup\n\n### Create Admin User\n\n```bash\nphp artisan tinker\n\n# In tinker console:\n$user = App\\Models\\User::create([\n    'name' => 'Admin',\n    'email' => 'admin@smmelite.com',\n    'password' => bcrypt('SecurePassword123'),\n    'is_admin' => true,\n    'status' => 'active',\n]);\n```\n\nAdmin panel: `https://yourdomain.com/admin`\n\n## API Provider Integration\n\n### Adding a Provider\n\n1. Go to Admin Panel → Providers\n2. Click \"Create Provider\"\n3. Enter:\n   - **Name**: Provider name (e.g., \"AMI SMM\")\n   - **URL**: API endpoint\n   - **API Key**: Your provider API key\n   - **Markup**: Percentage increase for pricing\n4. Click \"Create\"\n5. Click \"Sync\" to import services\n\n### Supported Provider Format\n\nProviders must respond with:\n\n```json\n{\n  \"services\": [\n    {\n      \"service\": \"1\",\n      \"name\": \"Instagram Followers\",\n      \"category\": \"Instagram\",\n      \"rate\": 0.5,\n      \"min\": 10,\n      \"max\": 100000\n    }\n  ]\n}\n```\n\n## Cron Jobs\n\nScheduled tasks run automatically:\n\n- **Every 5 minutes**: Sync order statuses from providers\n- **Daily at midnight**: Refresh exchange rates (USD to PKR)\n- **Hourly**: Process failed transactions\n\n## Environment Variables Reference\n\n```env\n# Application\nAPP_NAME=SMM Elite\nAPP_ENV=production|local\nAPP_DEBUG=false\nAPP_URL=https://yourdomain.com\nAPP_KEY=base64:...\n\n# Database\nDB_CONNECTION=mysql\nDB_HOST=localhost\nDB_PORT=3306\nDB_DATABASE=smmelite\nDB_USERNAME=username\nDB_PASSWORD=password\n\n# Mail\nMAIL_MAILER=smtp\nMAIL_HOST=smtp-relay.brevo.com\nMAIL_PORT=587\nMAIL_USERNAME=...\nMAIL_PASSWORD=...\nMAIL_FROM_ADDRESS=noreply@smmelite.com\n\n# Payment\nSTRIPE_PUBLIC_KEY=pk_live_...\nSTRIPE_SECRET_KEY=sk_live_...\nSTRIPE_WEBHOOK_SECRET=whsec_...\n\nPAYPAL_MODE=live\nPAYPAL_CLIENT_ID=...\nPAYPAL_SECRET=...\nPAYPAL_WEBHOOK_ID=...\n\n# Application Settings\nREFERRAL_COMMISSION_PERCENT=5\nMIN_DEPOSIT_AMOUNT=100\nMAX_DEPOSIT_AMOUNT=500000\nPAGINATION_PER_PAGE=20\n\n# Queue\nQUEUE_CONNECTION=database  # or redis\nREDIS_HOST=127.0.0.1\nREDIS_PASSWORD=null\nREDIS_PORT=6379\n\n# Cache\nCACHE_DRIVER=redis  # or file, array\n```\n\n## Troubleshooting\n\n### Database Connection Error\n\n```bash\n# Check database configuration\nphp artisan tinker\n# Test: DB::connection()->getPdo();\n```\n\n### Queue Not Processing\n\n```bash\n# Check queue status\nphp artisan queue:failed\n# Retry failed jobs\nphp artisan queue:retry all\n```\n\n### Asset Not Loading\n\n```bash\n# Rebuild assets\nnpm run build\n# Clear cache\nphp artisan view:clear\nphp artisan cache:clear\n```\n\n### Provider Sync Failing\n\n```bash\n# Check logs\nphp artisan logs\n# Test provider connection manually\nphp artisan tinker\n# $api = new App\\Services\\ProviderApiService($provider);\n# $api->getBalance();\n```\n\n## Security Best Practices\n\n1. **Change Default Keys**\n   - Generate new `APP_KEY`\n   - Update database credentials\n\n2. **Enable HTTPS**\n   - Set `APP_URL` to `https://`\n   - Update `SESSION_SECURE_COOKIES=true` in `.env`\n\n3. **Rate Limiting**\n   - Configured via `config/queue.php`\n   - Adjust `API_RATE_LIMIT` as needed\n\n4. **API Key Encryption**\n   - Provider API keys are encrypted\n   - Keep `APP_KEY` secure\n\n5. **Admin Protection**\n   - Use strong admin passwords\n   - Enable 2FA if possible\n   - Review admin action logs regularly\n\n6. **Input Validation**\n   - All forms use FormRequest validation\n   - User input is sanitized\n\n## Support & Updates\n\nFor issues or questions:\n1.
