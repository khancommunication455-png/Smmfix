# DEPLOYMENT GUIDE - SMM Elite\n\n## Quick Start Deployment\n\nThis project is production-ready and can be deployed to Railway in minutes.\n\n## Option 1: Railway Quick Deploy\n\n### Step 1: Prepare Repository\n\n```bash\n# Ensure all code is committed\ngit add .\ngit commit -m \"Ready for production deployment\"\ngit push origin main\n```\n\n### Step 2: Railway Setup\n\n1. Go to https://railway.app\n2. Sign in with GitHub\n3. Click \"New Project\" → \"Deploy from GitHub repo\"\n4. Select your SMM Elite repository\n5. Railway auto-detects `Procfile` and `nixpacks.toml`\n\n### Step 3: Add Services\n\nIn Railway dashboard:\n\n1. **Add MySQL Database**\n   - Click \"+ Add Service\"\n   - Select \"MySQL\"\n   - Configure: Version 8.0, 1GB RAM minimum\n\n2. **Add Redis (Optional but recommended)**\n   - Click \"+ Add Service\"\n   - Select \"Redis\"\n   - This enables fast caching and queue processing\n\n### Step 4: Configure Environment Variables\n\nIn Railway project settings, add environment variables:\n\n```env\nAPP_NAME=SMM Elite\nAPP_ENV=production\nAPP_DEBUG=false\nAPP_KEY=base64:<your-256-bit-key>\nAPP_URL=https://<your-domain>.railway.app\n\nDB_CONNECTION=mysql\nDB_HOST=${{mysql.MYSQL_HOST}}\nDB_PORT=${{mysql.MYSQL_PORT}}\nDB_DATABASE=${{mysql.MYSQL_DB}}\nDB_USERNAME=${{mysql.MYSQL_USER}}\nDB_PASSWORD=${{mysql.MYSQL_PASSWORD}}\n\nQUEUE_CONNECTION=redis\nREDIS_HOST=${{redis.REDIS_HOST}}\nREDIS_PASSWORD=${{redis.REDIS_PASSWORD}}\nREDIS_PORT=${{redis.REDIS_PORT}}\n\nCACHE_DRIVER=redis\nSESSION_DRIVER=cookie\n\nMAIL_MAILER=smtp\nMAIL_HOST=smtp-relay.brevo.com\nMAIL_PORT=587\nMAIL_USERNAME=<your-brevo-email>\nMAIL_PASSWORD=<your-brevo-password>\nMAIL_FROM_ADDRESS=noreply@smmelite.com\n\nSTRIPE_PUBLIC_KEY=pk_live_...\nSTRIPE_SECRET_KEY=sk_live_...\nSTRIPE_WEBHOOK_SECRET=whsec_...\n\nPAYPAL_MODE=live\nPAYPAL_CLIENT_ID=...\nPAYPAL_SECRET=...\n```\n\n### Step 5: Deploy\n\n1. Click \"Deploy\" button\n2. Wait for build to complete (2-5 minutes)\n3. You'll see 3 processes:\n   - `web` (serves the website)\n   - `worker` (processes background jobs)\n   - `scheduler` (runs cron jobs)\n\n### Step 6: Run Migrations\n\n```bash\n# In Railway terminal\nrailway run php artisan migrate --force\nrailway run php artisan db:seed --force\n```\n\nOr use Railway CLI:\n\n```bash\nrailway link <project-id>\nrailway run php artisan migrate --force\nrailway run php artisan db:seed --force\n```\n\n### Step 7: Configure Custom Domain\n\n1. In Railway project settings → Domains\n2. Click \"+ New Domain\"\n3. Enter your custom domain\n4. Update DNS CNAME record to Railway endpoint\n5. Update `APP_URL` in environment variables\n\n### Step 8: Verify Deployment\n\n```bash\n# Test application\ncurl https://<your-domain>/\n\n# Check admin login\nhttps://<your-domain>/login\n# Email: admin@smmelite.com\n# Password: (see DatabaseSeeder.php)\n```\n\n## Option 2: Manual Deployment to VPS\n\n### Requirements\n\n- Ubuntu 22.04 LTS (or equivalent)\n- 2GB RAM, 20GB SSD minimum\n- PHP 8.2, MySQL 8.0, Nginx\n- Composer, Node.js 18+\n\n### Installation Steps\n\n```bash\n# 1. Update system\nsudo apt update && sudo apt upgrade -y\n\n# 2. Install dependencies\nsudo apt install -y php8.2 php8.2-mysql php8.2-curl php8.2-zip \\\n    php8.2-intl php8.2-gd php8.2-redis php8.2-bcmath \\\n    mysql-server nginx redis-server supervisor nodejs npm git\n\n# 3. Clone repository\ngit clone <your-repo> /var/www/smmelite\ncd /var/www/smmelite\n\n# 4. Install dependencies\ncomposer install --no-dev --optimize-autoloader\nnpm ci && npm run build\n\n# 5. Configure .env\ncp .env.example .env\nphp artisan key:generate\n\n# Edit .env with your database and payment credentials\nnano .env\n\n# 6. Setup database\nmysql -u root -p < database/database.sql\nphp artisan migrate --force\nphp artisan db:seed --force\n\n# 7. Set permissions\nsudo chown -R www-data:www-data /var/www/smmelite\nsudo chmod -R 755 /var/www/smmelite\nsudo chmod -R 777 /var/www/smmelite/storage\n\n# 8. Setup Nginx\nsudo nano /etc/nginx/sites-available/smmelite\n# (Configure Nginx virtual host - see Nginx config below)\n\nsudo nginx -t\nsudo systemctl restart nginx\n\n# 9. Setup queue worker\nsudo nano /etc/supervisor/conf.d/smmelite-worker.conf\n# (Configure supervisor - see Supervisor config below)\n\nsudo supervisorctl reread\nsudo supervisorctl update\n\n# 10. Setup cron scheduler\n(crontab -l 2>/dev/null; echo \"* * * * * cd /var/www/smmelite && php artisan schedule:run >> /dev/null 2>&1\") | crontab -\n```\n\n### Nginx Configuration\n\nCreate `/etc/nginx/sites-available/smmelite`:\n\n```nginx\nserver {\n    listen 80;\n    server_name smmelite.com www.smmelite.com;\n    root /var/www/smmelite/public;\n\n    add_header X-Frame-Options \"SAMEORIGIN\" always;\n    add_header X-Content-Type-Options \"nosniff\" always;\n    add_header X-XSS-Protection \"1; mode=block\" always;\n\n    index index.html index.htm index.php;\n    charset utf-8;\n\n    location / {\n        try_files $uri $uri/ /index.php?$query_string;\n    }\n\n    location = /favicon.ico {\n        access_log off;\n        log_not_found off;\n    }\n\n    location = /robots.txt {\n        access_log off;\n        log_not_found off;\n    }\n\n    error_page 404 /index.php;\n\n    location ~ \\.php$ {\n        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;\n        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;\n        include fastcgi_params;\n    }\n\n    location ~ /\\.(?!well-known).*$ {\n        deny all;\n    }\n}\n```\n\nEnable site:\n\n```bash\nsudo ln -s /etc/nginx/sites-available/smmelite /etc/nginx/sites-enabled/\nsudo systemctl restart nginx\n```\n\n### Supervisor Configuration\n\nCreate `/etc/supervisor/conf.d/smmelite-worker.conf`:\n\n```ini\n[program:smmelite-worker]\nprocess_name=%(program_name)s_%(process_num)02d\ncommand=php /var/www/smmelite/artisan queue:work redis --sleep=3 --tries=3 --timeout=90\nautostart=true\nautorestart=true\nnumprocs=4\nredirect_stderr=true\nstdout_logfile=/var/www/smmelite/storage/logs/worker.log\nuser=www-data\n```\n\n### SSL Certificate (Let's Encrypt)\n\n```bash\nsudo apt install -y certbot python3-certbot-nginx\nsudo certbot certonly --nginx -d smmelite.com -d www.smmelite.com\n```\n\nUpdate Nginx to use SSL (HTTPS).\n\n## Database Backups\n\n### Railway Automated Backups\n\nRailway automatically backs up MySQL database. No action needed.\n\n### Manual Backup\n\n```bash\n# Export database\nmysqldump -u user -p database > backup.sql\n\n# Restore database\nmysql -u user -p database < backup.sql\n```\n\n## Monitoring & Logs\n\n### Railway Logs\n\n```bash\nrailway logs\n```\n\n### Application Logs\n\nView in Railway dashboard or:\n\n```bash\nrailway run tail -f storage/logs/laravel.log\n```\n\n### Database Logs\n\n```bash\nrailway run php artisan tinker\n# Check: Log::all()->latest()->first();\n```\n\n## Updating Application\n\n### Deploy New Version\n\n```bash\n# Commit changes\ngit add .\ngit commit -m \"Feature: xyz\"\ngit push\n\n# Railway auto-deploys\n# Or manually trigger in Railway dashboard\n```\n\n### Run Migrations After Update\n\n```bash\nrailway run php artisan migrate --force\n```\n\n## Troubleshooting\n\n### Worker Not Processing Jobs\n\n```bash\nrailway run php artisan queue:failed\nrailway run php artisan queue:retry all\n```\n\n### Database Connection Error\n\n```bash\nrailway run php artisan tinker\n# Test: DB::connection()->getPdo();\n```\n\n### View Errors\n\n```bash\nrailway logs\n# or\nrailway run tail -f storage/logs/laravel.log\n```\n\n## Performance Optimization\n\n### Enable Caching\n\n```bash\nphp artisan config:cache\nphp artisan route:cache\nphp artisan view:cache\n```\n\n### Database Optimization\n\n```bash\nphp artisan tinker\n# Run migrations with indexes\nDB::unprepared('OPTIMIZE TABLE orders;');\nDB::unprepared('OPTIMIZE TABLE users;');\nDB::unprepared('OPTIMIZE TABLE transactions;');\n```\n\n## Security Checklist\n\n- [ ] Change all default passwords\n- [ ] Set `APP_DEBUG=false`\n- [ ] Enable HTTPS/SSL\n- [ ] Configure firewall rules\n- [ ] Regular backups enabled\n- [ ] Logs monitored\n- [ ] Payment credentials secured\n- [ ] Admin 2FA enabled (if available)\n- [ ] Database user has limited privileges\n- [ ] composer.lock committed to version control\n\n## Support\n\nFor deployment issues:\n1. Check: `railway logs`\n2. Check: `storage/logs/laravel.log`\n3. Review: Error stack traces\n4. Check: Database connection\n5. Verify: Environment variables\n
